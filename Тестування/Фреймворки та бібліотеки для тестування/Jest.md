# Jest

Jest — це універсальний фреймворк для тестування JavaScript, який підтримує проекти, що використовують React, TypeScript, Node.js, Angular, Vue та інші технології. Він надає простий інтерфейс для написання тестів, має багатий набір функцій для тестування і пропонує різноманітні можливості, такі як паралельне виконання тестів і показ покриття коду тестами.

Основні особливості Jest:

-   Не потребує складного налаштування.
-   Має чудову документацію та активну спільноту користувачів.
-   Надає вбудовану бібліотеку тверджень для спрощення написання тестів.
-   Підтримує асинхронні тести та тести з функціями зворотного виклику.
-   Можливість генерації звітів про тестування та покриття коду.

## Налаштування `package.json`

У файлі package.json можна налаштувати сценарії (scripts) для виконання різних команд Jest.

```json
{
    "scripts": {
        "test": "jest",
        "test:watch": "jest --watch",
        "test:coverage": "jest --coverage"
    }
}
```

У цьому прикладі, команда `npm run test` запускає всі тести у вашому проекті за допомогою Jest. Команда `npm run test:watch` запускає Jest у режимі перегляду, де він автоматично перезапускає тести після змін у вашому коді. Команда `npm run test:coverage` виконує тести та генерує звіт про покриття коду тестами.

## Опції Jest CLI

-   `--onlyChanged` — ця опція дозволяє запустити лише змінені файли з вашого Git репозиторію, замість виконання всіх тестів.
-   `--testNamePattern=<regex>` та `--testPathPattern=<regex>` — за замовчуванням, Jest шукає файли тестів з певним патерном (`*.test.js` та `*.spec.js`) у всьому проекті. Проте, ви можете використовувати ці опції, щоб обмежити виконання тестів до файлів, що відповідають певному регулярному виразу. Це дозволяє вам розділити тести на групи та виконувати їх окремо.
-   `--watch` та `--watchAll` — ці опції дозволяють автоматично виконувати тести після кожної зміни файлу. `--watch` запускає тести лише для файлу, що змінився, а `--watchAll` запускає тести для всього проекту після кожної зміни.
