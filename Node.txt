Ресурс -- сущность яка потрібна клаєнту від нас
Через єнд поінт отримуємо доступ до ресурса
Стутус ответа

C - post
R - get
U - put/patch -- put {всі свойства}, patch {де-які свойства},
D - delete

http://localhost:3000/api/books/123 -- 123 URI parameters
http://localhost:3000/api/books?author=fowler&year=2022&limit=100&sort=title -- quary parameters,
можуть бути, а можуть і не бути

Якщо хочемо передати дані о даних, або щось допоміжне ми передаємо ці данні в headers


GraphQL

Призначення сервера GraphQL:
 - Повернення заповнених полів схеми як відповідь
 - Перевірка запитів GraphQL за нашою схемою
 - Отримання вхідних запитів GraphQL

Призначення резольверів:
 - Резолвер — це функція, яка відповідає за заповнення даних для одного поля вашої схеми
 - Резолвер перетворює дані у форму, яку вимагає запит
